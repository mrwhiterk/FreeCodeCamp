Introduction

JavaScript is a cross-platform, object-oriented scripting language. It is a small and lightweight language. Inside a host environment (for example, a web browser), JavaScript can be connected to the objects of its environment to provide programmatic control over them.

JavaScript contains a standard library of objects, such as Array, Date, and Math, and a core set of language elements such as operators, control structures, and statements. Core JavaScript can be extended for a variety of purposes by supplementing it with additional objects; for example:

  • Client-side JavaScript extends the core language by supplying objects to control a browser and its Document Object Model (DOM). For example, client-side extensions allow an application to place elements on an HTML form and respond to user events such as mouse clicks, form input, and page navigation.

  • Server-side JavaScript extends the core language by supplying objects relevant to running JavaScript on a server. For example, server-side extensions allow an application to communicate with a database, provide continuity of information from one invocation to another of the application, or perform file manipulations on a server.

JavaScript and Java

Beginners tend to assume that because JavaScript has “Java” in its name they are somehow related. While most programmers groan at this coupling and many even feel that the name confusion is part of a marketing gimmick, the history of the two programming languages did intersect for a brief moment in time during the early days of Netscape. The evolution of the two languages took such wildly different paths from that point forward.

Javascript is interpreted language meaning that it parsed at runtime into binaries that your computer can understand. It is also dynamically typed meaning that checking the types of variables (if you added type checking) will happen during runtime.

Java is a compiled language meaning that it needs to be converted into binaries before the code can be executed. It is also statically typed meaning that it will check for types and error even before the code is executed.

Hello World

"Hello World" is typically the first "program" you make when learning a new programming language.

You can print "Hello World" to the console in JavaScript with:
console.log('Hello World') // "Hello World"
console.log(typeof 'Hello World') // "string"

Variables

A variable is a named location for storing a value. That way an unpredictable value can be accessed through a predetermined name.

Example:
var name = 'Ryan'
Above, we are storing the string 'Ryan' in a variable we called 'name'.

Declaring Variables

The are three ways to declare variables in JavaScript: var, let, const

Example:
var name = 'Ryan';
Above, we are storing the string 'Ryan' in a variable we called 'name'.

let num = 5;
Above, we are storing the number 5 in a variable we called 'num'.

const PI = 3.14;
Above, we are storing the number 3.14 in a constant we called 'PI'. Note that constants are typically typed in all uppercase to remind the user that this value should not be changed.

Variable Scope

JavaScript has two scopes – global and local. Any variable declared outside of a function belongs to the global scope, and is therefore accessible from anywhere in your code. Each function has its own scope, and any variable declared within that function is only accessible from that function and any nested functions.

Example:
function doSomething () {
let name = 'Ryan';
}
doSomething()
console.log(name)
ReferenceError: name is not defined

Above, we are creating a function and declaring a variable called name inside of it. After, we are invoking that function and then we try to print the name variable. However, received a Reference error because name is in a scope that you do not have access to globally.
